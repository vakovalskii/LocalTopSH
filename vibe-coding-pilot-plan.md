# План запуска Vibe Coding пилота

## Неделя 0 — Согласование и запуск

- [ ] Получить бюджет на подписки: 4 человека × $180 = $720/мес
- [ ] Определить конкретный сервис для MVP+ из бэклога (что-то понятное и нужное, не research)
- [ ] Сформировать пилотную группу: 1 лид + 2-3 разработчика, готовых работать в новом формате
- [ ] Развернуть инструменты: Claude Code Max, Cursor Pro
- [ ] Настроить единые `.cursorrules` и `CLAUDE.md` по кодовой базе

## Недели 1-2 — Разработка MVP+ в режиме vibe coding

- [ ] Группа работает по единому своду правил: промптинг, code review агентного кода, структура проекта
- [ ] Лид кодит вместе с командой, не только управляет — это принципиально
- [ ] Ежедневные 15-минутные синки: что сработало, что нет, какие промпты дали результат
- [ ] Параллельно: маппинг существующих бизнес-процессов компании для Трека 2 — поиск кандидатов на автоматизацию

## Недели 3-4 — Результат и демо

- [ ] Демо MVP+ для руководства: сервис, сроки, как это было сделано
- [ ] Зафиксировать факты:
  - Сколько дней заняло vs. классическая оценка
  - Сколько кода сгенерировано агентами
- [ ] По Треку 2: подготовить 2-3 конкретных процесса компании с оценкой эффекта от автоматизации (не абстракции, а конкретика с цифрами)

## Неделя 5+ — Масштабирование или pivot

**Если демо зашло:**
- Расширяем группу
- Подключаем второй сервис
- Запускаем первый пилот по автоматизации процессов

**Если не зашло:**
- У нас есть данные почему
- Потрачено $720, а не полгода бюджета на "классический диджитал"
